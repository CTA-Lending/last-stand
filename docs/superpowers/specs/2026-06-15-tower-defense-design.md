## 1. 願景一句話

一款在瀏覽器打開就能玩、畫風精緻（扁平向量 + 發光粒子特效）的**高奇幻固定路徑塔防**。
頭牌玩法是同名的**無盡生存模式「誰撐最久」**——怪物程式生成、一波比一波強，比拼存活時間與波數，直上世界排行榜；
另有戰役關卡模式。核心技巧建立在「攻擊類型 × 護甲類型」克制表上，外掛**轉蛋抽塔、角色圖鑑、每日登入**的長線養成。

## 3. 遊戲模式

### 3.1 無盡生存模式「誰撐最久」（頭牌）
- 怪物**程式生成、隨機組成**，從傳送門不斷湧入。
- **一波比一波強**：隨波數放大怪的血量、數量、移速、賞金；每隔 N 波出精英/首領。
- 玩家邊撐邊蓋塔/升級/放主動技，命歸零即結束。
- **計分 = 存活波數 + 存活時間**（漏怪有寬限命數）。
- 結束上傳「撐最久榜」（一期本機 + 模擬全球；二期真世界榜）。
- 程式生怪邏輯集中於 `EndlessDirector` 模組，難度曲線參數化於 `balance.js`。

### 3.2 戰役關卡模式
- 3 張手工設計地形，固定波次、末波首領。
- 難度：普通／英雄／地獄，記錄最佳通關時間與分數。

## 4. 核心玩法循環

1. 選模式（無盡 / 戰役）→ 選地圖與難度。
2. 進場前編「出戰陣容」：基礎塔永遠可帶，外加圖鑑中已解鎖的轉蛋塔。
3. 怪物沿固定路徑走向**王城**，漏一隻扣命（起始 20 命）。
4. 殺怪掉金 → 場上蓋塔／升級／放主動技 → 無盡模式撐越久越好 / 戰役撐過全波過關。
5. 結束記錄成績（波數＋時間 / 通關時間）→ 排行榜 / 本機最佳。
6. 領每日登入券 → 轉轉樂抽塔 → 擴充圖鑑與陣容。

## 5. 技巧系統（遊戲靈魂）

### 5.1 克制表：攻擊類型 × 護甲類型

| 攻擊＼護甲 | 輕甲（快速兵） | 重甲（坦克） | 法甲（術士） | 飛行 |
|---|---|---|---|---|
| 物理（弓） | ×1.5 | ×0.5 | ×1.0 | 可打 |
| 攻城（砲） | ×1.0 | ×1.5 | ×0.75 | 不可打 |
| 魔法（法師） | ×1.0 | ×1.0 | ×1.5 | 可打 |

- 額外非傷害手段：**減速/定身**（冰/纏繞）、**持續毒 DoT**（無視護甲，剋高血首領）。
- 倍率為初版設定，集中於資料檔，不寫死在引擎。

### 5.2 塔的鎖定優先序
每座塔可切換：最前／最後／最強（最高血量）／最近。手動切換是進階技巧。

### 5.3 主動技（操作技巧，不需控英雄）
- 🔥 **火雨**：點地施放範圍爆發，冷卻長。
- ❄️ **寒冰術**：全場敵人短暫凍結，救場用。

## 6. 守護方完整塔陣容（6 陣營 × 3 塔）

每座塔 3 階升級，頂階分支成 2 種專精。

| 陣營 | 塔 1 | 塔 2 | 塔 3 |
|---|---|---|---|
| 🏹 精靈 | 神射手（物理單體/打空）→ 連射 / 狙擊暴擊 | 月刃塔（彈射多體） | 纏繞德魯伊（減速/定身） |
| ⚒️ 矮人 | 蒸汽火砲（攻城濺射/不打空）→ 爆裂 / 燃燒 | 遠程投石（長射程 AoE） | 地雷工程（路徑陷阱） |
| ⚔️ 人類 | 騎士兵營（出兵**擋路**近戰）| 十字弩（物理穿透/打空） | 號令旗（範圍 buff 友塔） |
| 🔮 魔法師 | 奧術塔（魔法單體/剋法甲/打空）| 連環閃電（跳躍多體） | 變形術（機率秒變羊） |
| 🐉 龍族 | 幼龍塔（火焰錐 AoE/打空）*傳奇高費* | 烈焰祭壇（高傷 DoT） | — |
| ✨ 神族 | 聖殿光環（全場增益）*終極解鎖* | 審判落雷（真傷 AoE） | — |

### 特殊機制
- **人類騎士兵營**：在集結點生成近戰士兵，**擋住路徑上的敵人**並纏鬥；陣亡後冷卻重生。卡位 + 集結點選擇是核心技巧。
- **龍族 / 神族**：定位為轉蛋傳奇塔，需從圖鑑解鎖才能帶上場，造價/冷卻高、威力大。

### 塔的兩種來源
- **基礎塔**（精靈/矮人/魔法師/人類角色定位塔）：永遠可用，保證克制系統完整、隨時能玩。
- **轉蛋/收藏塔**（稀有變體 + 龍/神傳奇 + 英雄塔）：轉轉樂抽出後進圖鑑，編入出戰陣容方可使用。

每怪有：血量、護甲類型、移速、賞金、特殊能力。全部走資料檔，供無盡模式隨機抽取與縮放。

## 8. Meta 養成系統

### 8.1 轉轉樂（Gacha）
- 每日登入領 1 張轉券；亦可用遊戲內貨幣兌換。
- 轉盤/開卡演出 → 抽出塔卡，稀有度 R／SR／SSR／UR。
- 重複 → 碎片 → 升星（提升屬性）。機率表透明、集中於資料檔。

### 8.2 角色圖鑑（角色設定）
每座塔 = 角色卡：向量立繪、名字、背景故事、陣營、稀有度、屬性、性格、升星等級。圖鑑可翻閱。

### 8.3 每日登入
連續登入獎勵；至少每日 1 轉券。記錄 last login 於本機存檔。

### 8.4 出戰陣容
進場前從圖鑑選 5–6 張塔組 loadout；基礎塔預設可選。

## 9. 排行榜 / 計時 / 難度

- **撐最久榜（頭牌）**：無盡模式存活波數 + 時間排名 → 呼應遊戲名。
- **戰役榜**：每關 ×（普通/英雄/地獄）最佳通關時間、分數、難度對比視圖。
- **一期**：全存本機，排行榜顯示「本機最佳 + 模擬全球榜」（種子假資料）。
- **二期**：接 VPS 後端，提交真實成績、抓全球前段班。以 `LeaderboardService` 介面抽象（一期 local 實作、二期 remote 實作），不動遊戲本體。

---

## 10. 關卡 / 地形

- 無盡模式：可在任一地圖開跑（首發至少 1 張無盡專用地圖）。
- 戰役模式：3 張手工地形（精靈森林 → 矮人山道 → 魔界之門），難度遞增、地形差異（單路徑 → 分岔 → 多傳送門），各約 15–20 波末波首領。
- 地圖以資料定義：tile 網格、waypoint 路徑折線、建塔格、集結點。架構支援無限擴充。

## 11. 技術架構

### 11.1 技術選型
- HTML5 **Canvas 2D** + 原生 JS（**ES Modules**），零框架依賴。
- 跑法：一行本機 http 伺服器（ES Modules 需 http，例 `python -m http.server`），日後可打包單檔。
- 渲染分層（下→上）：地形 → 建塔格 → 塔 → 怪 → 投射物 → 粒子 → （DOM）UI。
- UI 用 DOM/HTML 疊在 canvas 上（建造選單、金錢/命、波次、技能鈕、結算、圖鑑、轉蛋、排行榜）。

### 11.2 模組切分（單一職責、可獨立測試）
- `core/loop`：requestAnimationFrame 固定時間步更新 + 渲染。
- `core/pathing`：依預存 waypoint 折線移動，無 A*。
- `systems/waveManager`：戰役依資料排程生怪。
- `systems/endlessDirector`：無盡模式程式生怪 + 難度縮放。
- `systems/economy`：金錢、命數、分數、計時。
- `systems/combat`：克制表結算、DoT、減速。
- `systems/targeting`：範圍查詢 + 鎖定優先序。
- `entities/`：tower、enemy、projectile、soldier（兵營出兵）、effect。
- `render/`：分層渲染器 + 粒子系統（物件池）。
- `meta/`：gacha、collection、dailyLogin、loadout。
- `services/`：`SaveService`（localStorage，含版本/遷移）、`LeaderboardService`（介面 + local/remote 實作）。
- `ui/`：DOM 疊層各畫面。
- `data/`：towers、enemies、waves、maps、gachaTable、balance。全資料驅動。

### 11.3 存檔（本機）
localStorage 存：圖鑑解鎖與升星、貨幣與轉券、每日登入時間、各模式各關各難度最佳成績、設定。以 `SaveService` 封裝含版本號。

### 11.4 效能
固定路徑 + 中量實體，Canvas 2D 足夠。粒子用物件池；離屏快取靜態地形；空間查詢用網格分格。特效/實體量暴增再評估 PixiJS（WebGL）。

## 12. 分階段開發節奏（先有得玩，再長肉）

- **Phase 1 — 無盡生存垂直切片（同名招牌玩法）**：引擎（loop/pathing/render/combat/targeting/economy）、`endlessDirector` 程式生怪與難度縮放、1 張地圖、精靈/矮人/魔法師 3 座基礎塔（含升級）、不死族數種怪、命數與存活波數/時間計分、本機最佳記錄、基礎粒子與打擊感、戰鬥 HUD。**目標：驗證好不好玩。**
- **Phase 2 — 內容長肉**：補滿 6 陣營全 18 塔與升級分支、人類兵營擋路、魔族陣容、主動技（火雨/寒冰）、第 2 張地圖。
- **Phase 3 — 養成 + 戰役模式**：轉轉樂、角色圖鑑與角色設定、每日登入、出戰陣容、龍/神傳奇塔、戰役關卡與難度、過關計時、模擬全球榜、開卡/轉盤演出。
- **Phase 4 — 封頂與上線**：第 3 張地圖、首領戰打磨、精緻特效與打擊感調校、平衡；接 VPS 後端開**真・世界撐最久榜**（二期）。

每個 Phase 結束都有可玩、可驗收的版本。

## 13. 非目標 / YAGNI（首發不做）

- 不做帳號/登入認證（一期本機即可）。
- 不做付費金流（轉券靠每日登入與遊戲產出）。
- 不做多人即時對戰。
- 不做手繪美術資產（全向量程式繪製）。
- 不做 A* 動態尋路（固定路徑足矣）。
- 不做造迷宮玩法。

## 14. 風險與待解

- **平衡數值**：克制倍率、塔造價、怪血量、無盡難度曲線需大量實戰調校 → 全集中於 `balance.js`。
- **向量畫風的精緻感**：靠發光/粒子/動畫補足，Phase 1 就建好粒子與打擊感基礎。
- **轉蛋與基礎塔平衡**：傳奇塔不可碾壓基礎塔，定位為特定解法。
- **存檔遷移**：`SaveService` 帶版本號。

## 15. Phase 4 大改造：養成 RPG 塔防（2026-06-15 CEO 定案）

從「隨時自由選關的休閒塔防」轉型為「線上遊戲式的養成 RPG 塔防」。

### 核心循環
**大廳(Lobby)** → 選副本(模式/地圖/難度)＋編隊 → **進入副本** → 遊玩 → **結算(得鑽石)** → 回大廳。
進入副本後地圖/模式不可中途切換（修掉「中途點地圖會重開」的 bug）。

### 兩種貨幣
- 💰 **金幣**：局內貨幣，每局重置，關卡內蓋塔/升級（維持現狀經濟）。
- 💎 **鑽石**：帳號貨幣，**破關才取得**，去**商城**買**永久塔**。

### 帳號養成（RPG）
- **擁有的塔** = 起始 4 基礎塔(精靈神射手/矮人火砲/魔法師奧術/騎士兵營) ＋ 商城鑽石買的永久塔 ＋ 轉蛋抽到的稀有塔。
- **編隊(loadout)**：進副本前從擁有的塔挑一套帶入；關卡內只能蓋帶入的塔。
- **科技樹 = 帳號路線**：每個帳號在商城解鎖的塔組合不同 → 高豐富度。
- 移除舊的「場上要有前置塔才能蓋」in-match 限制 → 改為編隊制。

### 名詞
- **副本** = 闖關（情慾關卡 × 難度）；破關依難度給鑽石。
- **帳號** = RPG 養成（鑽石/擁有塔/編隊）。

### 敵方陣營：七情六慾（敘事核心，取代不死/魔族）
玩家＝正義一方，使命是**淨化/滅掉具象化的心魔**——七情與六慾。小怪＝該情慾的化身/心魔碎片，Boss＝情慾本體。

**七情篇（7 關，喜怒哀懼愛惡欲）** — 每關一個情 Boss，專屬機制：
- 😤怒(狂暴，被打加速)、😢哀(悲傷光環降周圍塔攻速)、😱懼(讓塔偶爾失準)、😍愛(召喚護衛)、😈惡(高血坦，死亡散腐蝕)、😀喜(治療激勵其他心魔)、🤑欲(碰到偷金幣)

**六慾篇（6 關，色聲香味觸意）** — 六感之慾，精英/小 Boss：
- 色慾(魅惑)、聲慾(音波)、香慾(毒霧)、味慾(吞噬)、觸慾(纏繞減速)、意慾(幻象分身)

**關卡結構**：七情篇 7 關 + 六慾篇 6 關 = 13 關，分兩大章，難度遞增、每關一情慾 Boss。

### 分階段
- 4a：大廳/結算循環（修跳地圖 bug、結束才結算回大廳）。✅
- 4b：鑽石＋商城＋帳號擁有塔＋編隊（移除場上前置）。
- 4c：七情六慾敵方陣營（13 情慾 Boss＋專屬機制＋化身小怪，取代不死/魔族）。
- 4d：副本關卡＝七情篇 7 關＋六慾篇 6 關（章節地圖、破關給鑽石）。
- 4e：科技樹視覺化＋平衡。
- 最後：視覺打磨 Pass。
